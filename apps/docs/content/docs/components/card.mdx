## Styling

### Passing Tailwind CSS classes

```tsx
import { Card } from '@heroui/react';

function CustomCard() {
  return (
    <Card className="border-2 border-blue-500 bg-gradient-to-r from-blue-50 to-purple-50">
      <Card.Header>
        <Card.Title>Custom Styled Card</Card.Title>
      </Card.Header>
    </Card>
  );
}
```

### Customizing the component classes

To customize the Card component classes, you can use the `@layer components` directive.
<br/>[Learn more](https://tailwindcss.com/docs/adding-custom-styles#adding-component-classes).

```css
@layer components {
  .card {
    @apply shadow-lg border-2 bg-gradient-to-br from-white to-gray-50;
  }

  .card--surface-2 {
    @apply bg-gradient-to-br from-blue-50 to-purple-50;
  }

  .card__title {
    @apply text-xl font-bold text-blue-900;
  }
}
```

HeroUI follows the [BEM](https://getbem.com/) methodology to ensure component variants and states are reusable and easy to customize.

### CSS Classes

The Card component uses these CSS classes ([View source styles](https://github.com/heroui-inc/heroui/blob/v3/packages/styles/components/card.css)):

#### Base Classes
- `.card` - Base card container
- `.card__header` - Header section
- `.card__title` - Title element
- `.card__description` - Description text
- `.card__content` - Main content area
- `.card__details` - Details container (for variants)
- `.card__footer` - Footer section
- `.card__image` - Image element
- `.card__close-button` - Close button

#### Variant Classes
- `.card--side` - Horizontal layout variant
- `.card--item` - Compact layout variant
- `.card--full` - Overlay layout variant
- `.card--surface-none` - Transparent background surface
- `.card--surface-1` - Default surface level
- `.card--surface-2` - Medium surface level
- `.card--surface-3` - High surface level
- `.card__image--side` - Image styling for side variant
- `.card__image--item` - Image styling for item variant
- `.card__image--full` - Image styling for full variant
- `.card__header--full` - Header styling for full variant
- `.card__details--side` - Details styling for side variant
- `.card__details--item` - Details styling for item variant
- `.card__details--full` - Details styling for full variant
- `.card__content--full` - Content styling for full variant
- `.card__footer--item` - Footer styling for item variant
- `.card__footer--full` - Footer styling for full variant

### Interactive States

The card close button supports both CSS pseudo-classes and data attributes for flexibility:

- **Hover**: `:hover` or `[data-hover="true"]`
- **Active/Pressed**: `:active` or `[data-pressed="true"]` (includes scale transform)
- **Focus**: `:focus-visible` or `[data-focus-visible="true"]` (shows focus ring)
- **Disabled**: `:disabled` or `[aria-disabled="true"]` (reduced opacity, no pointer events)

## API Reference

### Card Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'side' \| 'item' \| 'full'` | - | Layout variant for different card styles |
| `surface` | `'none' \| '1' \| '2' \| '3'` | `'1'` | Surface elevation level for visual hierarchy |
| `asChild` | `boolean` | `false` | Render as a child element |
| `className` | `string` | - | Additional CSS classes |
| `children` | `React.ReactNode` | - | Card content |

### Card.Header Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `asChild` | `boolean` | `false` | Render as a child element |
| `className` | `string` | - | Additional CSS classes |
| `children` | `React.ReactNode` | - | Header content |

### Card.Title Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `asChild` | `boolean` | `false` | Render as a child element (default: h3) |
| `className` | `string` | - | Additional CSS classes |
| `children` | `React.ReactNode` | - | Title content |

### Card.Description Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `asChild` | `boolean` | `false` | Render as a child element (default: p) |

| `children` | `React.ReactNode` | - | Description content |

### Card.Content Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|

### Card.Footer Props

| Prop | Type | Default | Description |

| `className` | `string` | - | Additional CSS classes |

### Card.Image Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `src` | `string` | - | Image source URL |
| `alt` | `string` | - | Alternative text for accessibility |
| `asChild` | `boolean` | `false` | Render as a child element (default: img) |
| `className` | `string` | - | Additional CSS classes |

### Card.CloseButton Props

Inherits all props from React Aria's `Button` component:

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `onPress` | `(e: PressEvent) => void` | - | Handler called when button is pressed |
| `isDisabled` | `boolean` | `false` | Whether the button is disabled |
| `asChild` | `boolean` | `false` | Render as a child element |
| `className` | `string` | - | Additional CSS classes |
| `children` | `React.ReactNode \| (values: ButtonRenderProps) => React.ReactNode` | `<CloseIcon />` | Button content or render prop |
