## Overview

HeroUI v3 provides a flexible theming system built on CSS variables and [BEM](https://getbem.com/) class naming. You can customize the entire design system through CSS, from colors and spacing to component-specific styles.

HeroUI's theming heavily relies on [Tailwind CSS v4's theming strategies](https://tailwindcss.com/docs/theme) and [custom styling approaches](https://tailwindcss.com/docs/adding-custom-styles), leveraging features like CSS layers, theme variables, and the `@theme` directive for a powerful and flexible customization experience.


## Design Tokens

HeroUI includes `light` and `dark` themes. Apply them by adding a theme class to `<html>` (or root) and color utilities to `<body>`:

```html
<html className="light" data-theme="light">
  <body className="bg-background text-foreground">
    <!-- Your app content -->
  </body>
</html>
```

### Color System

HeroUI's color system aligns with web standards:

- **Accent Color**: Following the [browser standard](https://developer.mozilla.org/en-US/docs/Web/CSS/accent-color), `--accent` serves as the primary brand color for interactive elements
- **Base/Gray Colors**: The neutral color palette provides the foundation for UI elements
- **Surface Levels**: Three surface variables (`--surface-1`, `--surface-2`, `--surface-3`) create visual hierarchy through depth

## Understanding Color Naming

### Accent as Primary

HeroUI uses `--accent` as the primary brand color, following the [CSS accent-color](https://developer.mozilla.org/en-US/docs/Web/CSS/accent-color) standard. This provides semantic alignment with native browser controls:

```css
/* Your primary brand color */
--accent: oklch(0.7 0.25 260);
--accent-foreground: var(--white);

/* Used in buttons, links, and interactive elements */
.button--primary {
  background: var(--accent);
  color: var(--accent-foreground);
}
```

### Surface Hierarchy

The surface variables create visual depth and content hierarchy:

```css
--surface-1: var(--background);      /* Page background level */
--surface-2: var(--color-neutral-100); /* Cards, panels */
--surface-3: var(--color-neutral-200); /* Nested content, tooltips */
```

Use surfaces to create visual separation:
- `surface-1`: Base level, typically the page background
- `surface-2`: Elevated content like cards and panels
- `surface-3`: Highest elevation for popovers, tooltips, and modals

## Customizing Variables

### Global Override

Override variables in your root CSS file:

```css
/* app.css */
@import "tailwindcss";
@import "@heroui/styles";

:root {
  /* Override theme colors */
  --accent: oklch(0.7 0.25 260);
  --background: #fafafa;
  --foreground: #0a0a0a;
  
  /* Adjust spacing */
  --spacing: 0.5rem;
  
  /* Custom radius */
  --radius: 0.5rem;
}
```

### Dark Mode

HeroUI includes dark mode variables:

```css
/* Automatic dark mode */
@media (prefers-color-scheme: dark) {
  :root {
    --background: var(--black);
    --foreground: var(--snow);
    --panel: var(--eclipse);
  }
}

/* Class-based dark mode */
.dark {
  --background: var(--black);
  --foreground: var(--snow);
  --accent: var(--color-neutral-50);
  --border: oklch(1 0 0 / 12%);
}
```

### Custom Theme

Create your own theme:

```css
/* themes/ocean.css */
.ocean,
[data-theme="ocean"] {
  /* Ocean theme colors */
  --background: oklch(0.98 0.01 210);
  --foreground: oklch(0.15 0.02 210);
  --panel: oklch(0.96 0.015 210);
  
  /* Accent colors */
  --accent: oklch(0.6 0.2 210);
  --accent-foreground: var(--white);
  --accent-soft: oklch(0.92 0.04 210);
  --accent-soft-foreground: oklch(0.3 0.15 210);
  
  /* Surface colors - layered depth */
  --surface-1: var(--background);
  --surface-2: oklch(0.94 0.02 210);
  --surface-3: oklch(0.90 0.03 210);
  
  /* Default colors */
  --default: oklch(0.95 0.01 210);
  --default-foreground: oklch(0.4 0.05 210);
  
  /* Muted and scrollbar */
  --muted: oklch(0.55 0.08 210);
  --scrollbar: oklch(0.85 0.04 210);
  
  /* Status colors */
  --success: oklch(0.7 0.15 150);
  --success-foreground: var(--white);
  --warning: oklch(0.75 0.15 45);
  --warning-foreground: var(--black);
  --danger: oklch(0.65 0.2 15);
  --danger-foreground: var(--white);
  
  /* Borders & Focus */
  --border: oklch(0.6 0.05 210 / 15%);
  --focus: oklch(0.6 0.2 210 / 60%);
  --link: oklch(0.55 0.18 210);
  
  /* Radius - slightly rounded for modern ocean feel */
  --radius: 1rem;
  --radius-panel: 1.25rem;
  --radius-panel-inner: calc(var(--radius-panel) * 0.6);
  
  /* Shadows - soft ocean-inspired */
  --shadow-border: 
    0 0px 8px 0px oklch(0.5 0.1 210 / 0.08),
    0 4px 12px 0px oklch(0.5 0.1 210 / 0.12),
    0 0px 1px 0px oklch(0.5 0.1 210 / 0.25);
}
```

Import and use your custom theme:

```css
/* global.css */
@layer theme, base, components, utilities;

@import "tailwindcss";
@import "@heroui/styles";

/* Import custom theme */
@import "./themes/ocean.css" layer(theme);
```

## Global Component Styles

Override component styles globally using `@layer components`:

```css
/* global.css */
@layer components {
  /* Button customization */
  .button {
    @apply font-semibold tracking-wide;
  }
  
  .button--primary {
    @apply bg-blue-600 hover:bg-blue-700;
  }
  
  /* Accordion customization */
  .accordion {
    @apply border-2 border-gray-200;
  }
  
  .accordion__trigger {
    @apply text-lg font-bold;
  }
}
```

- **BEM classes** (e.g., `.button`, `.button--primary`, `.accordion__trigger`)
- **Interactive states** (e.g., `[data-hover="true"]`, `[aria-expanded="true"]`)

## Import Strategies

### Full Import (Default)

Import everything for a complete styled experience:

```css
@import "tailwindcss";
@import "@heroui/styles";
```

### Selective Import

Import only what you need for smaller bundles using Tailwind CSS v4 layer syntax:

```css
/* Define layers first */
@layer theme, base, components, utilities;

/* Base Tailwind */
@import "tailwindcss";

/* Animations (optional) */
@import "tw-animate-css";

/* Core variables and base styles */
@import "@heroui/styles/base/base.css" layer(base);
@import "@heroui/styles/themes/default.css" layer(theme);

/* Import all components at once */
@import "@heroui/styles/components/index.css" layer(components);

/* OR import individual components */
@import "@heroui/styles/components/button.css" layer(components);
@import "@heroui/styles/components/accordion.css" layer(components);
@import "@heroui/styles/components/avatar.css" layer(components);
@import "@heroui/styles/components/chip.css" layer(components);
@import "@heroui/styles/components/link.css" layer(components);
@import "@heroui/styles/components/popover.css" layer(components);
@import "@heroui/styles/components/spinner.css" layer(components);
@import "@heroui/styles/components/tooltip.css" layer(components);

/* Optional utilities */
@import "@heroui/styles/utilities/index.css" layer(utilities);
/* OR individual utilities */
@import "@heroui/styles/utilities/backdrop.css" layer(utilities);
```

**Note**: 
- The `layer()` syntax ensures proper CSS cascade and specificity
- Import `tw-animate-css` if you need animations, or omit it for smaller bundle size
- Choose between importing all components via `index.css` or individual component files

### Headless Mode

For completely unstyled components, import only the base:

```css
@import "tailwindcss";
@import "@heroui/styles/base/base.css";

/* Build your own component styles */
.button {
  /* Your custom button styles */
}
```

## Best Practices

### 1. Use Semantic Variables

```css
/* Good: Semantic meaning */
--accent: #5B21B6;
--danger: #DC2626;

/* Avoid: Color names */
--purple: #5B21B6;
--red: #DC2626;
```

### 2. Layer Your Overrides

```css
/* 1. Import HeroUI */
@import "@heroui/styles";

/* 2. Override variables */
:root {
  --accent: #5B21B6;
}

/* 3. Component customization */
@layer components {
  .button {
    text-transform: uppercase;
  }
}
```

### 3. Maintain Consistency

Use the same variables across your app:

```css
/* Define once */
:root {
  --brand-radius: 0.375rem;
}

/* Use everywhere */
.button { border-radius: var(--brand-radius); }
.card { border-radius: var(--brand-radius); }
.input { border-radius: var(--brand-radius); }
```

### 4. Document Your Theme

```css
/* Theme: Ocean Blue
 * Primary: Deep blue for main actions
 * Secondary: Teal for accents
 * Status: Green success, amber warning
 */
.ocean, 
[data-theme="ocean"] {
  --accent: oklch(0.5 0.2 240);       /* Deep blue */
  --accent-foreground: oklch(0.95 0.01 210);
  --accent-soft: oklch(0.92 0.04 210);
  --accent-soft-foreground: oklch(0.3 0.15 210);

  --surface-1: oklch(0.94 0.02 210);
  --surface-2: oklch(0.90 0.03 210);
  --surface-3: oklch(0.85 0.04 210);

  --secondary: oklch(0.7 0.15 180);   /* Teal */
  --secondary-foreground: oklch(0.95 0.01 210);
  --success: oklch(0.7 0.2 150);      /* Green */
  --success-foreground: oklch(0.95 0.01 210);
  --warning: oklch(0.75 0.18 60);     /* Amber */
  --warning-foreground: oklch(0.95 0.01 210);
}
```
