## Philosophy

### Styled by Default with Tailwind CSS v4

HeroUI v3 components come with beautiful default styles powered by Tailwind CSS v4. Components are ready to use out of the box:

```tsx
import { Button } from '@heroui/react';

// Renders a styled button with HeroUI's design system
<Button>Click me</Button>
```

### Built on CSS Variables

The design system uses CSS variables for easy customization:

```css
:root {
  --accent: #3b82f6;
  --background: #ffffff;
  --foreground: #000000;
  --radius: 0.5rem;
}
```

### Tailwind CSS v4 Required

HeroUI v3 is built on top of Tailwind CSS v4. This is not optional - Tailwind CSS provides the foundation for all component styles.

## CSS Setup

Ensure your CSS file imports both Tailwind CSS and HeroUI styles:

```css
/* src/index.css */
@import "tailwindcss";
@import "@heroui/styles";
```

**Important**: Always import `tailwindcss` before `@heroui/styles`.

## Styling Approaches

### 1. CSS Classes

The most straightforward approach using CSS classes:

```css
/* styles.css */
.my-button {
  background: #3b82f6;
  color: white;
  padding: 0.5rem 1rem;
  border-radius: 0.375rem;
  border: none;
  cursor: pointer;
  font-weight: 500;
}

.my-button:hover {
  background: #2563eb;
}

.my-button:focus-visible {
  outline: 2px solid #3b82f6;
  outline-offset: 2px;
}
```

```tsx
import { Button } from '@heroui/react';
import './styles.css';

function MyButton() {
  return (
    <Button className="my-button">
      Click me
    </Button>
  );
}
```

### 2. Tailwind CSS Classes

Override default styles with Tailwind CSS utilities:

```tsx
import { Button } from '@heroui/react';

function CustomButton() {
  return (
    <>
      {/* Use default HeroUI styling */}
      <Button>Default Button</Button>
      
      {/* Override with custom Tailwind classes */}
      <Button className="bg-purple-500 hover:bg-purple-600">
        Custom Purple Button
      </Button>
    </>
  );
}
```

### 3. CSS Modules

Type-safe styling with CSS Modules:

```css
/* Button.module.css */
.button {
  background: #3b82f6;
  color: white;
  padding: 0.5rem 1rem;
  border-radius: 0.375rem;
  border: none;
  cursor: pointer;
  font-weight: 500;
}

.primary {
  background: #3b82f6;
}

.secondary {
  background: #6b7280;
}

.large {
  padding: 0.75rem 1.5rem;
  font-size: 1.125rem;
}
```

```tsx
import { Button } from '@heroui/react';
import styles from './Button.module.css';
import { clsx } from 'clsx';

interface StyledButtonProps {
  variant?: 'primary' | 'secondary';
  size?: 'default' | 'large';
  children: React.ReactNode;
}

function StyledButton({ variant = 'primary', size = 'default', children }: StyledButtonProps) {
  return (
    <Button 
      className={clsx(
        styles.button,
        styles[variant],
        size === 'large' && styles.large
      )}
    >
      {children}
    </Button>
  );
}
```

### 4. Styled Components

CSS-in-JS with styled-components:

```tsx
import { Button } from '@heroui/react';
import styled from 'styled-components';

const StyledButton = styled(Button)<{ variant?: 'primary' | 'secondary' }>`
  background: ${props => props.variant === 'secondary' ? '#6b7280' : '#3b82f6'};
  color: white;
  padding: 0.5rem 1rem;
  border-radius: 0.375rem;
  border: none;
  cursor: pointer;
  font-weight: 500;
  
  &:hover {
    background: ${props => props.variant === 'secondary' ? '#4b5563' : '#2563eb'};
  }
  
  &:focus-visible {
    outline: 2px solid ${props => props.variant === 'secondary' ? '#6b7280' : '#3b82f6'};
    outline-offset: 2px;
  }
`;

function MyComponent() {
  return (
    <div>
      <StyledButton variant="primary">Primary</StyledButton>
      <StyledButton variant="secondary">Secondary</StyledButton>
    </div>
  );
}
```

## BEM CSS Integration

HeroUI v3 provides optional BEM-style CSS classes for consistent styling:

```css
/* Optional: Import HeroUI's BEM utilities */
@import "@heroui/styles/components/button.css";

/* Use BEM classes */
.button {
  /* Base button styles */
}

.button--primary {
  /* Primary variant */
}

.button--large {
  /* Large size */
}
```

```tsx
import { Button, buttonVariants } from '@heroui/react';

function BEMButton() {
  return (
    <Button className={buttonVariants({ variant: 'primary', size: 'large' })}>
      BEM Styled Button
    </Button>
  );
}
```

Style individual parts of compound components:

```tsx
import { Alert } from '@heroui/react';

function StyledAlert() {
  return (
    <Alert className="border border-yellow-200 bg-yellow-50 p-4 rounded-md">

        <Alert.Description className="text-yellow-700 mt-1">

        </Alert.Description>

```

## State-Based Styling

Style components based on their state using data attributes:

```css
.my-button {
  background: #3b82f6;
  transition: all 0.2s;
}

.my-button[data-pressed="true"] {
  transform: scale(0.95);
}

.my-button[data-disabled="true"] {
  opacity: 0.5;
  cursor: not-allowed;
}

.my-button[data-focus-visible="true"] {
  outline: 2px solid #3b82f6;
  outline-offset: 2px;
}
```

```tsx
import { Button } from '@heroui/react';

function StatefulButton() {
  return (
    <Button 
      className="my-button"
      isDisabled={false}
    >
      Click me
    </Button>
  );
}
```

## Design System Integration

### Creating a Design System

Build reusable styled components for your design system:

```tsx
// components/Button.tsx
import { Button as HeroButton, type ButtonProps } from '@heroui/react';
import { cva, type VariantProps } from 'class-variance-authority';

const buttonVariants = cva(
  // Base styles
  "inline-flex items-center justify-center rounded-md font-medium transition-colors focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-offset-2 disabled:pointer-events-none disabled:opacity-50",
  {
    variants: {
      variant: {
        primary: "bg-blue-500 text-white hover:bg-blue-600 focus-visible:ring-blue-500",
        secondary: "bg-gray-100 text-gray-900 hover:bg-gray-200 focus-visible:ring-gray-500",
        outline: "border border-gray-300 bg-transparent hover:bg-gray-50 focus-visible:ring-gray-500",
      },
      size: {
        sm: "h-9 px-3 text-sm",
        md: "h-10 px-4",
        lg: "h-11 px-8 text-lg",
      },
    },
    defaultVariants: {
      variant: "primary",
      size: "md",
    },
  }
);

export interface CustomButtonProps 
  extends ButtonProps, 
    VariantProps<typeof buttonVariants> {}

export function Button({ 
  className, 
  variant, 
  size, 
  ...props 
}: CustomButtonProps) {
  return (
    <HeroButton
      className={buttonVariants({ variant, size, className })}
      {...props}
    />
  );
}
```

## Best Practices

### 1. Consistent Naming

Use consistent class naming conventions:

```css
/* Good */
.button { }
.button--primary { }
.button--large { }

/* Avoid */
.btn { }
.primaryButton { }
.large-button { }
```

### 2. Component Composition

Style compound components as cohesive units:

```tsx
function Card({ children }: { children: React.ReactNode }) {
  return (
    <div className="bg-white rounded-lg shadow-md overflow-hidden">
      {children}
    </div>
  );
}

function CardHeader({ children }: { children: React.ReactNode }) {
  return (
    <div className="px-6 py-4 border-b border-gray-200">
      {children}
    </div>
  );
}

function CardContent({ children }: { children: React.ReactNode }) {
  return (
    <div className="px-6 py-4">
      {children}
    </div>
  );
}
```

### 3. Accessible Styling

Ensure your styles support accessibility:

```css
.button {
  /* Ensure sufficient color contrast */
  background: #1f2937; /* Dark enough for white text */
  color: white;
  
  /* Clear focus indicators */
  focus-visible:outline: 2px solid #3b82f6;
  focus-visible:outline-offset: 2px;
  
  /* Don't rely on color alone */
  border: 2px solid transparent;
}

.button[aria-pressed="true"] {
  border-color: #3b82f6; /* Visual indicator beyond color */
}
```

### 4. Performance

Optimize for performance:

```css
/* Use CSS custom properties for dynamic values */
.button {
  background: var(--button-bg, #3b82f6);
  transition: background-color 0.2s; /* Only animate necessary properties */
}

/* Avoid expensive operations */
.button:hover {
  /* Good: simple color change */
  background: #2563eb;
  
  /* Avoid: expensive transforms on hover */
  /* transform: scale(1.05); */
}
```

## Theming

### CSS Custom Properties

Create themeable components with CSS custom properties:

```css
.theme-light {
  --bg-primary: #ffffff;
  --text-primary: #1f2937;
  --border-color: #e5e7eb;
}

.theme-dark {
  --bg-primary: #1f2937;
  --text-primary: #f3f4f6;
  --border-color: #374151;
}

.card {
  background: var(--bg-primary);
  color: var(--text-primary);
  border: 1px solid var(--border-color);
}
```

```tsx
function ThemedCard() {
  const [theme, setTheme] = useState('light');
  
  return (
    <div className={`theme-${theme}`}>
      <Card className="card">
        <CardContent>
          This card adapts to the theme!
        </CardContent>
      </Card>
    </div>
  );
}
```
